# 01. 랭플로우(Langflow)란?

## 개요

Langflow는 **오픈소스, Python 기반의 비주얼 AI 애플리케이션 프레임워크**입니다. 복잡한 코드를 작성하지 않고도 드래그 앤 드롭 방식으로 AI 워크플로우를 설계하고, 테스트하고, 배포할 수 있습니다.

### 주요 특징

- **비주얼 에디터**: 드래그 앤 드롭으로 AI 워크플로우를 구성
- **LLM 비종속**: 특정 LLM이나 벡터 스토어에 종속되지 않음
- **실시간 테스트**: Playground에서 즉시 플로우를 테스트
- **API 배포**: 만든 플로우를 REST API로 바로 배포 가능
- **에이전트 지원**: 자율적으로 도구를 선택하고 실행하는 AI 에이전트 구축
- **MCP(Model Context Protocol) 지원**: MCP 서버와의 연동

---

## 핵심 개념

### 1. Flow (플로우)

**플로우**는 AI 애플리케이션의 전체 워크플로우를 의미합니다. 하나의 플로우는 여러 컴포넌트들이 연결된 형태로 구성됩니다.

```
[Chat Input] → [Prompt] → [LLM] → [Chat Output]
```

플로우는 Langflow의 비주얼 에디터(Workspace)에서 생성하고 관리합니다.

### 2. Component (컴포넌트)

**컴포넌트**는 플로우를 구성하는 개별 단위입니다. 각 컴포넌트는 워크플로우의 한 단계를 담당합니다.

컴포넌트의 종류:
- **범용 컴포넌트**: Inputs, Outputs, Data, Prompts 등
- **전문 컴포넌트**: Agents, Language Models, Embedding Providers 등
- **번들 컴포넌트**: 외부 서비스(OpenAI, Pinecone 등)와 연동하는 컴포넌트

### 3. Node (노드)

비주얼 에디터에서 컴포넌트가 화면에 배치된 형태를 **노드**라고 합니다. 각 노드는:
- **입력 포트(Port)**: 다른 노드로부터 데이터를 받는 연결점
- **출력 포트(Port)**: 다른 노드로 데이터를 보내는 연결점
- **설정 필드**: 컴포넌트의 동작을 구성하는 파라미터

### 4. Edge (엣지)

**엣지**는 노드와 노드를 연결하는 선입니다. 엣지를 통해 데이터가 한 컴포넌트에서 다른 컴포넌트로 흘러갑니다. 포트의 색상이 같은 것끼리 연결할 수 있으며, 색상은 데이터 타입의 호환성을 나타냅니다.

### 5. Workspace (워크스페이스)

**워크스페이스**는 플로우를 만드는 메인 작업 공간입니다. 이곳에서:
- 컴포넌트를 드래그 앤 드롭으로 추가
- 컴포넌트 간 연결(엣지) 생성
- 각 컴포넌트의 설정 조정

### 6. Playground (플레이그라운드)

**플레이그라운드**는 만든 플로우를 실시간으로 테스트할 수 있는 환경입니다.
- 플로우와 대화 형식으로 상호작용
- 각 컴포넌트의 입출력 확인
- LLM의 메모리를 수정하여 응답 튜닝

## 전체 흐름 이해하기

```
┌─────────────────────────────────────────────────┐
│                  Langflow UI                     │
│                                                  │
│  ┌──────────┐    ┌──────────┐    ┌──────────┐  │
│  │Chat Input│───▶│  Prompt  │───▶│   LLM    │  │
│  └──────────┘    └──────────┘    └──────────┘  │
│                                       │          │
│                                       ▼          │
│                                ┌──────────┐      │
│                                │Chat Output│     │
│                                └──────────┘      │
│                                                  │
│  [Playground에서 테스트] → [API로 배포]           │
└─────────────────────────────────────────────────┘
```

1. **설계**: Workspace에서 컴포넌트를 배치하고 연결
2. **테스트**: Playground에서 실시간으로 대화하며 검증
3. **배포**: API 엔드포인트로 외부 앱에서 호출
